# 숭산텍(SIT) 자율주행 프로젝트 개요

## 1. 개요 (Executive Summary)

숭산텍(SoongSanTech)의 자율주행 프로젝트는 **"엣지 디바이스 제약 하의 Sim-to-Real 경량 자율주행 시스템 구현 및 체계적 실증 분석"**이라는 단일 명제를 중심으로 구성된 소규모 연구 프로젝트입니다.

2인 팀이라는 현실적 제약을 명확히 인식하고, 새로운 아키텍처 발명보다는 **완전한 파이프라인 구축과 체계적 실증 실험**에 집중하는 방향성을 취하고 있습니다. 상용화 트랙과 연구 트랙을 순차적으로 배치하여 실용성과 학술적 기여를 동시에 추구합니다.

## 2. 5대 핵심 방향성 원칙

숭산텍의 기술 로드맵에는 다음 5가지 방향성 원칙이 명시되어 있으며, 이는 단순한 선언이 아니라 코드 구조와 실험 설계에 직접 반영되어 있습니다.

### 2.1 Sim-First 원칙
> "모든 학습과 검증은 CARLA에서 먼저 수행한다."

CARLA 0.9.15를 유일한 학습·검증 환경으로 채택하고, 실차(RC카) 배포는 시뮬레이터 검증이 완료된 이후로 명시적으로 후순위에 배치합니다. 실제 차량 테스트의 비용과 위험을 제거하면서도, 다양한 날씨·시간대 조합을 통해 환경 다양성을 확보하는 실용적 전략입니다.

### 2.2 BC → RL 2단계 학습 전략
> "행동복제로 초기 정책 확보 후 강화학습 미세조정."

단순한 알고리즘 선택이 아니라, 코드 수준에서 BC 모델의 ResNet18 backbone과 FC Head를 RL Actor Head로 직접 재사용하는 방식으로 구현됩니다. 초기 정책을 안정적으로 확보한 뒤, PPO로 미세조정하는 검증된 접근법입니다.

### 2.3 시스템 우선(System-First) 원칙
> "새로운 아키텍처 발명 배제, 검증된 모델(ResNet18) 채택."

ResNet18 + 224×224 입력을 고정 아키텍처로 채택한 것은 Jetson에서 30Hz 추론이라는 엣지 제약을 전제로 한 의도적 선택입니다. 연구의 초점을 "어떤 모델이 더 좋은가"가 아니라 "완전한 파이프라인에서 어떤 변수가 성능을 결정하는가"로 이동시킵니다.

### 2.4 엣지 최적화 원칙
> "모든 모델은 Jetson 30Hz 추론을 전제로 설계한다."

양자화 파이프라인은 `PyTorch(.pth) → ONNX → TensorRT FP16 → (선택) INT8` 순서로 설계되어 있으며, FP16에서 정확도 오차 2% 이내, INT8에서 5% 이내를 목표로 합니다. 추론 지연 목표는 Jetson 기준 10ms 미만입니다.

### 2.5 실증 연구 집중 원칙
> "모델 개발 대신, 완전한 파이프라인 기반 체계적 실험."

상용화 트랙 완료 후, 구축된 Sim-to-Real 루프를 활용하여 학계가 요구하는 희소한 실증 데이터를 산출하는 연구 트랙(Phase 3)을 별도로 설계하고 있습니다.

## 3. 프로젝트 페이즈 (Phases)

프로젝트는 크게 상용화 트랙(Phase 1, 2)과 연구 트랙(Phase 3)으로 나뉩니다.

- **Phase 1: Data Pipeline (완료)**
  - CARLA 멀티카메라 데이터 수집 파이프라인 구축
  - 장애 복원 및 10Hz 동기화 수집
- **Phase 2: ML Modeling & Productization (진행 중)**
  - Phase 2-A: ResNet18 기반 BC 모델 학습
  - Phase 2-B: PPO 기반 RL 미세조정
  - Phase 2-C: TensorRT 양자화 및 ROS2 엣지 배포
- **Phase 3: Experiments (예정)**
  - Phase 3-A: 데이터 양(Quantity) vs 다양성(Diversity) 실증
  - Phase 3-B: BC-to-RL 전이 효율성 실증 (Negative Transfer)
  - Phase 3-C: 양자화 안전성 분석 (Degradation Profile)
