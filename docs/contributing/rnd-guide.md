# R&D 로그 작성 가이드

## 1. R&D 로그의 목적

R&D 로그는 **연구 개발의 모든 의사결정과 실험 과정을 시계열로 투명하게 기록**하는 공간입니다. 위키가 "결론"을 다룬다면, R&D 로그는 "과정"을 다룹니다.

### 1.1 왜 모든 것을 기록해야 하는가

| 이유 | 설명 |
|------|------|
| 재현성 | 6개월 후에도 동일한 결과를 재현할 수 있어야 함 |
| 협업 | 팀원이 시도한 방법을 중복으로 시도하지 않도록 함 |
| 학습 | 실패 사례도 후속 연구의 귀중한 데이터 |
| 논문화 | Phase 3 실증 연구의 기반 자료 |

## 2. R&D 로그 카테고리

| 카테고리 | 용도 |
|---------|------|
| Experiments | 모델 학습, 하이퍼파라미터 탐색 결과 |
| Milestones | 주차/Phase별 목표 달성 회고 |
| Troubleshooting | 발생한 문제와 해결 과정 |
| Design Decisions | 아키텍처/기술 선택의 근거와 대안 비교 |

## 3. 파일명 규칙

```
docs/rnd/posts/YYYY-MM-DD-{슬러그}.md
```

예시:
- `2026-04-22-phase1-pipeline.md`
- `2026-05-03-bc-resnet18-first-training.md`
- `2026-05-10-troubleshooting-carla-timeout.md`

## 4. Front Matter 표준

```yaml
---
title: "[Phase 2-A] ResNet18 BC 모델 첫 학습 결과"
date: 2026-05-03
categories:
  - R&D
  - Experiments
tags:
  - bc
  - resnet18
  - phase2-a
authors:
  - sit  # docs/rnd/.authors.yml에 정의된 ID
---
```

`categories`는 사이드바 그룹화에, `tags`는 검색·필터링에 사용됩니다.

## 5. 실험 로그 표준 구조

```markdown
## 1. 개요 및 목표
- 실험 가설 (H1, H2, ...)
- 검증하고자 하는 것

<!-- more -->  ← 이 위가 미리보기로 노출됩니다.

## 2. 실험 환경
| 항목 | 값 |
|------|---|
| Git Commit | abc1234 |
| 데이터셋 | data/2026-04-30_*/ (총 3시간) |
| 하드웨어 | RTX 4090 |
| 시드 | 42 |

## 3. 하이퍼파라미터
(전체 설정을 명시)

## 4. 결과
- 학습 곡선 (그래프 첨부)
- 검증 메트릭 표
- 정성적 관찰

## 5. 분석 및 해석

## 6. 다음 단계 (Next Steps)
```

## 6. 트러블슈팅 로그 표준

```markdown
## 1. 증상
어떤 명령을 실행했을 때 어떤 에러가 발생했는지.

## 2. 원인 분석
재현 단계, 로그 발췌, 관련 코드 위치.

## 3. 해결 방법
적용한 수정 사항(코드 다이프, 설정 변경).

## 4. 교훈
같은 문제를 예방하기 위한 체크리스트.
```

## 7. Excerpt(미리보기) 작성

블로그 인덱스 페이지에는 `<!-- more -->` 마커 위쪽 내용이 미리보기로 노출됩니다. **3~4문장 이내로 핵심을 요약**하여 독자가 클릭할지 결정할 수 있도록 합니다.
